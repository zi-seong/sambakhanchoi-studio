# Code Style

## 기준

이 저장소의 코드 스타일은 클린 코드 원칙을 우선한다.

## 핵심 원칙

- 읽는 사람이 빠르게 이해할 수 있는 코드가 우선이다.
- 짧은 코드보다 의미가 분명한 코드를 선호한다.
- 한 함수와 한 컴포넌트는 하나의 중심 책임만 가져야 한다.
- 로직 중복을 줄이되, 성급한 공통화는 피한다.

## 네이밍

- 이름만 보고 역할을 짐작할 수 있어야 한다.
- `data`, `item`, `temp`, `handleStuff` 같은 모호한 이름은 피한다.
- boolean은 `is`, `has`, `can`, `should` 형태를 우선한다.
- 이벤트 핸들러는 `handle...` 형태를 사용한다.

## 함수

- 가능하면 입력과 출력이 분명해야 한다.
- 인자가 많아지면 객체 인자를 고려한다.
- 부작용이 있는 함수는 이름이나 위치로 드러나야 한다.
- 예외 상황은 조기에 반환해서 중첩을 줄인다.

## React 컴포넌트

- UI 렌더링, 상태 관리, 데이터 접근이 한 파일에 과도하게 몰리지 않게 한다.
- 재사용 가치가 있는 로직은 훅이나 저장소 계층으로 분리한다.
- 페이지 컴포넌트는 조합과 흐름 제어에 집중한다.
- 공통 UI는 일관된 props 인터페이스를 유지한다.

## TypeScript

- `any` 사용은 지양한다.
- 데이터 구조는 타입으로 먼저 표현한다.
- 외부 입력은 신뢰하지 말고 방어적으로 다룬다.
- 선택 속성은 정말 선택적일 때만 사용한다.

## 스타일링

- 기존 디자인 톤을 깨지 않는 범위에서 수정한다.
- 반복되는 색상, 간격, 패턴은 일관되게 유지한다.
- 클래스가 과도하게 길어지면 구조 분리를 먼저 검토한다.

## 에러 처리

- 실패는 조용히 삼키지 않는다.
- 사용자에게 영향을 주는 실패는 메시지로 드러낸다.
- 내부 로직 실패는 원인 추적이 가능하도록 작성한다.
